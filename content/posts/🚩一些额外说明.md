## Ubuntu分区说明（50G硬盘）

| 挂载点         | 建议大小               | 文件系统  | 说明                                 |
| ----------- | ------------------ | ----- | ---------------------------------- |
| `/boot/efi` | **512 MB**         | FAT32 | **必须存在**。UEFI 启动分区，安装程序通常会自动识别创建。  |
| `/`(根分区)    | **剩余全部 (约 49 GB)** | EXT4  | 包含系统、软件和你的个人文件（/home 也在其中）。        |
| `swap`      | **使用交换文件**         | -     | **无需单独分区**。系统安装后会自动创建 `/swapfile`。 |
## Ubuntu命令：

Ubuntu 使用 `apt`命令管理软件包。在分好区并进入系统后，常用命令如下：

|操作|命令示例|说明|
|---|---|---|
|**更新源**|`sudo apt update`|获取软件包列表的最新版本信息（安装系统后建议先执行）|
|**安装软件**|`sudo apt install vim git`|安装指定的软件包（如 vim、git）|
|**卸载软件**|`sudo apt remove vim`|卸载软件包（保留配置文件）|
|**彻底卸载**|`sudo apt purge vim`|彻底删除软件包及其配置文件|
|**搜索软件**|`apt search package_name`|在仓库中搜索包含关键字的软件包|

💽 如果是 `partitions`（查看分区结果）

安装完成后，可以通过以下命令验证 50G 硬盘的分区使用情况：

```Bash
# 查看磁盘分区布局（直观看到 / 分区是否占满 49G）
df -h

# 查看更详细的磁盘信息（包括未挂载的 swapfile）
lsblk
```
## 🔄 Git中裸仓库 vs 普通仓库

|特性|裸仓库（`--bare`）|普通仓库（无 `--bare`）|
|---|---|---|
|**工作区**|❌ 没有工作目录（没有 `.git`文件夹，整个目录就是 `.git`）|✅ 有工作目录，文件可直接编辑|
|**用途**|服务器端，仅用于代码同步|开发端，用于日常编码|
|**可推送**|✅ 允许 `git push`|❌ 默认拒绝 `git push`（需特殊配置）|
|**目录结构**|直接包含 `branches/`、`objects/`、`refs/`等|包含 `.git/`子目录和工作文件|
|**示例**|`project1.git/`（目录内是 Git 对象）|`project1/`（目录内是你的源代码文件）|

**查看对比**：

```Bash
# 创建对比示例
git init --bare bare-repo.git        # 裸仓库
git init normal-repo                 # 普通仓库

# 查看目录结构
ls -la bare-repo.git/                # 直接看到 hooks/, objects/, refs/, ...
ls -la normal-repo/                  # 看到 .git/ 目录和可能的文件
```
